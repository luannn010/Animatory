PRE-PRODUCTION

Starts from a final script. Three tracks run in parallel — Design, Storyboard, Casting — then converge at a "Checking" gate before shipping.

Showrunner / OrchestratorPRODUCTION ROLE: Producer · Script Breakdown
ORCHESTRATOR LLM
Owns the master plan. Reads the final script, produces the asset "breakdown," spawns the three parallel tracks, and gates every handoff.

Key Tasks

  • Parse script into scenes, beats, dialogue
  • Generate breakdown: every character, prop, vehicle, background
  • Assign work to track agents & track dependencies
  • Decide what's omitted/added after animatic

Decisions it makes

  • What needs designing this episode
  • When a track is "done enough" to hand off
  • Retry / revise vs. approve
Inputsfinal_script.txt
Outputsbreakdown.json (asset list + scene graph)
Design Agent — Line ArtPRODUCTION ROLE: B&W Rough → Final Models
IMAGE-GEN
Turns the breakdown into clean black-and-white model sheets — every character, prop, and background, drawn from multiple angles so a rig can be built.

Key Tasks

  • Rough predesign line art (feeds storyboard)
  • Refine to final clean line art
  • Character turnarounds (all angles)
  • Prop & BG sheets w/ scale reference

Must-haves

  • Consistent character identity across sheets
  • Show every angle (incl. "bottom of the shoe")
  • Multiple views = rig-ready
Inputsbreakdown.json, style refs
Outputsmodel_sheets/*.png (B&W line)
Design Agent — Color & BG PaintPRODUCTION ROLE: Color Styling · BG Paint · Final Color
IMAGE-GEN
Adds color styling to characters/props and paints backgrounds — producing day/night/lighting variants the vendor will color-match against.

Key Tasks

  • Color-style characters & props
  • Paint backgrounds (wide, high-res 2K/4K)
  • Generate lighting variants (day, night, sunset)
  • Lock final color palettes

Must-haves

  • Palette consistency per asset
  • Broad views so vendor can push in / reframe
Inputsmodel_sheets/*.png
Outputscolor_sheets/, bg_paint/
Storyboard Agent + AnimaticPRODUCTION ROLE: SB Roughs → Cleans → Editor → Animatic
LLM + IMAGE-GEN
Visualizes the script into shots and key poses, then an editor sub-step strings them into a timed animatic you can watch before any real animation.

Key Tasks

  • Plan shots, compositions, comedic beats
  • Draw rough key poses → clean them
  • Edit drawings to a timed sequence
  • Iterate on director/exec notes → final animatic

Must-haves

  • Key poses only (vendor fills in-betweens)
  • Camera-move + timing annotations
Inputsscript, predesign line art
Outputsanimatic.mp4 + boards/
Casting & Dialogue AgentPRODUCTION ROLE: Casting · Record · Dialogue Edit
AUDIO / TTS
Breaks the script down for voices, generates/records each character's dialogue, then cleans it (removing clicks, smacks, breaths).

Key Tasks

  • Assign a voice profile per character
  • Generate/record line reads
  • Dialogue edit: strip clicks & breaths
  • Produce dialogue-exposed X-sheets (timing)

Must-haves

  • Consistent voice per character
  • Timing data for lip-sync downstream
Inputsscript (dialogue lines)
Outputsdialogue/*.wav + xsheets.json
Mouth-Chart / Phoneme AgentPRODUCTION ROLE: 2D Mouth Charts · Expressions
IMAGE-GEN
Builds the mouth-shape library — one drawing per phoneme, per emotion — so the animator can sync mouths to dialogue even without understanding the language.

Key Tasks

  • Draw a mouth shape per phoneme (A,B,C,D…)
  • Variants per emotion (neutral/happy/sad/angry)
  • Map phonemes ↔ frames on the X-sheet

Must-haves

  • Language-agnostic lip-flap mapping
  • Aligns to dialogue timing
Inputscolor_sheets/, xsheets.json
Outputsmouth_charts/ + phoneme map
Checking / QA Gate AgentPRODUCTION ROLE: Checking · Final Materials Ship
QC / UTILITY
The convergence gate. Re-breaks-down the animatic vs. designs, confirms nothing is missing or extra, then packages everything for the vendor.

Key Tasks

  • Cross-check animatic against asset list
  • Flag omitted (stop work) / added (start work) assets
  • Validate completeness of every package
  • Bundle & ship final materials

Must-haves

  • "Vendor only knows what you send" — completeness is everything
  • Collaborative, multi-check (not one reviewer)
Inputsall design + audio + animatic + X-sheets
Outputsvendor_package.zip

PRODUCTION (Vendor Studio)

The package becomes real, moving animation. Builds rigs, fills in-betweens, syncs mouths — then sends "Take 1s" back for review loops.

Rigging AgentPRODUCTION ROLE: Rig Build (Harmony / Flash)
VIDEO / MOTION
Assembles each character into a reusable "puppet" — a library of poses, hands, and expressions from the turnarounds — so animation doesn't redraw everything.

Key Tasks

  • Build rig from multi-angle model sheets
  • Wire in hand poses & facial expressions
  • Attach color (already built into Harmony rig)
  • Set up scene files w/ all rigs loaded

Must-haves

  • All angles present or rig fails
  • Reusable pose library
Inputsvendor_package.zip
Outputsrigs/ + scene files
Animation Agent (Key + In-Betweens)PRODUCTION ROLE: Physical Animation · Lip-Sync
VIDEO / MOTION
Recreates the boards as real motion: blocks character action, draws the in-between frames for smooth movement, and animates lip-flap to the dialogue.

Key Tasks

  • Block action & camera moves from boards
  • Clean key poses, draw all in-betweens
  • Animate mouths via mouth-chart + X-sheet
  • Avoid "puppety" snapping between poses

Must-haves

  • Smooth motion paths (no pose-snaps)
  • Lip-flap matches recorded dialogue
Inputsrigs/, animatic, mouth_charts/, dialogue/
Outputstake1_shots/*.mp4 (colored)
Editor & Retake-Loop AgentPRODUCTION ROLE: Take 1s · Editor · Retakes
ORCHESTRATOR LLM
Receives finished shots ("Take 1s"), cuts them into a sequence, judges timing/quality, and loops retakes back to animation until it's right.

Key Tasks

  • Assemble shots into a cut
  • Check timing, clarity, asset correctness
  • Call retakes (loop back N times)
  • Emit the "1st Cut" when approved

Decisions

  • Approve vs. retake (budget/time-bounded)
  • Which shots are "less than desirable"
Inputstake1_shots/
Outputsfirst_cut.mp4 (+ retake requests)

POST-PRODUCTION

From the 1st cut, three audio tracks + an FX pass run in parallel, then everything locks, mixes, and ships.

ADR / Final Dialogue AgentPRODUCTION ROLE: Dial. Editor · ADR Spot · Record
AUDIO / TTS
Spots the cut for any changed, added, or re-performed lines, re-generates them, and edits to final clean dialogue.

Key Tasks

  • Spot lines to re-record (ADR)
  • Re-generate / re-read those lines
  • Edit out pops, clicks, breaths

Must-haves

  • Sync to locked picture timing
Inputsfirst_cut.mp4
Outputsfinal_dialogue.wav
Composer AgentPRODUCTION ROLE: Music Spot · Final Music
AUDIO (MUSIC)
Spots the cut for where music goes and what it should do (source vs. score, big vs. small, comedic), then composes the final music.

Key Tasks

  • Music spot: placement & intent per scene
  • Decide source music vs. wall-to-wall score
  • Compose & revise on notes

Must-haves

  • Tone matches comedy/emotion of scene
Inputsfirst_cut.mp4
Outputsfinal_music.wav
SFX House AgentPRODUCTION ROLE: SFX Spot · Final SFX
AUDIO (SFX)
Spots every non-vocal sound the picture needs (doors, punches, whooshes), then sources from a library or generates them fresh.

Key Tasks

  • SFX spot: list every needed effect
  • Pull from library or synthesize new
  • Review placement & fit

Must-haves

  • Frame-accurate placement
Inputsfirst_cut.mp4
Outputsfinal_sfx.wav
After-FX / VFX AgentPRODUCTION ROLE: After FX Spot · Final FX
VIDEO / MOTION
"Plusses" the picture — upgrading simple drawn effects (a plain dust cloud) into particles, glows, and richer composited visual effects.

Key Tasks

  • Spot shots needing FX plus-ups
  • Add particles / glows / composites
  • Review & finalize FX

Must-haves

  • Matches show's visual style
Inputsfirst_cut.mp4
Outputsfinal_fx_shots/
Mix · Color · Delivery AgentPRODUCTION ROLE: Picture Lock · Mix · Color Correction · Online · QC/CC · Deliver
QC / UTILITY
The final merge. Locks picture once credits + all elements are in, mixes audio to balance, color-corrects, then does online/QC/captioning and ships.

Key Tasks

  • Assemble credits, confirm all elements → picture lock
  • Mix: balance dialogue under music & SFX
  • Optional color correction / vignettes
  • Online edit, QC, closed captioning
  • Deliver to network / streamer

Must-haves

  • Dialogue always audible over music
  • First moment everything is seen together

END-TO-END AGENT FLOW

Script→ Orchestrator→ Design ‖ Storyboard ‖ Casting→ Checking Gate→ Rig → Animate→ Editor / Retakes→ Dialogue ‖ Music ‖ SFX ‖ VFX→ Mix → Deliver

Building this as a real agent flow — 4 things to get right

1. Parallel + gated. Pre-pro and post-pro both fan out into parallel tracks, then re-converge at a gate (Checking, Editor, Mix). Your orchestrator should spawn tracks concurrently and block on the gate, not run everything linearly.

2. Consistency is the hard problem. Image and video agents must keep the same character identity, palette, and rig across every shot. Lock a style/character reference early and pass it to every downstream visual agent — this is the AI equivalent of the model sheets.

3. The "vendor only knows what you send" rule. Each agent's output package must be self-contained. An animation agent can't infer the bottom of a shoe — if the design agent didn't produce that view, generation breaks. Validate completeness at the Checking gate before any expensive video generation runs.

4. Retake loops cost the most. The Editor→Animation loop and post-pro reviews are where time/money go. Put cheap LLM/QC review before expensive video-gen, and cap retake iterations explicitly.
